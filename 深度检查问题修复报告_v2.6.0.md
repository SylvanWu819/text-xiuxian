### 潜在问题1: requiredItems数组中重复itemId的处理

**位置**: `src/game/OptionSystem.ts` checkRequirements方法

**问题描述**:
```javascript
if (option.requirements.requiredItems) {
  for (const itemId of option.requirements.requiredItems) {
    if (this.resourceManager.hasItem(itemId)) {
      continue;
    }
    return false;
  }
}
```

如果`requiredItems: ["spirit_herb", "spirit_herb"]`（需要2个灵草），当前代码只检查是否拥有至少1个，而不是检查数量。

**建议修复**:
```javascript
if (option.requirements.requiredItems) {
  // 统计每个道具需要的数量
  const itemCounts = new Map();
  for (const itemId of option.requirements.requiredItems) {
    itemCounts.set(itemId, (itemCounts.get(itemId) || 0) + 1);
  }
  
  // 检查是否拥有足够数量
  for (const [itemId, requiredCount] of itemCounts.entries()) {
    if (!this.resourceManager.hasItem(itemId, requiredCount)) {
      return false;
    }
  }
}
```

**状态**: ⚠️ 需要修复

### 潜在问题2: itemChanges负数消耗时的错误处理

**位置**: `src/game/GameEngine.ts` applyEffects方法

**问题描述**:
```javascript
if (change < 0) {
  const removed = this.resourceManager.removeItem(itemId, Math.abs(change));
  if (removed) {
    negativeEffects.push(`📦 消耗道具：${itemId} ×${Math.abs(change)}`);
  }
}
```

如果`removeItem`返回false（道具不足），代码不会显示任何消息，用户不知道发生了什么。

**建议修复**:
```javascript
if (change < 0) {
  const removed = this.resourceManager.removeItem(itemId, Math.abs(change));
  if (removed) {
    negativeEffects.push(`📦 消耗道具：${itemId} ×${Math.abs(change)}`);
    messages.push(`你消耗了道具：${itemId} ×${Math.abs(change)}。`);
  } else {
    // 道具不足，这不应该发生（因为选项需求已检查）
    console.error(`[GameEngine] 道具不足，无法消耗：${itemId} ×${Math.abs(change)}`);
    messages.push(`⚠️ 道具不足，无法消耗：${itemId}。`);
  }
}
```
