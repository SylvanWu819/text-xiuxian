# UI改进计划 v2.6.0

## 改进目标

根据游戏内容指南和用户反馈，优化UI界面，提升用户体验。

---

## 1. 背包UI完善

### 当前状态
- ✅ 背包数据结构已实现
- ✅ 背包在详细面板中显示
- ✅ 道具名称中文映射
- ⚠️ UI展示需要优化

### 改进方案

#### 1.1 背包图标和分类
```css
/* 道具分类图标 */

}

.item-pill::before { content: '💊'; }
.item-material::before { content: '🌿'; }
.item-artifact::before { content: '⚔️'; }
.item-special::before { content: '✨'; }
```

#### 1.2 道具详情提示
- 鼠标悬停显示道具详细信息
- 包括：名称、类型、效果、获取方式

#### 1.3 道具排序
- 按类型分组显示
- 按稀有度排序
- 按数量排序

## 4. 选项系统优化

### 4.1 选项状态指示
```css
/* 可选 */
.option-available {
  border-left: 3px solid #4ec9b0;
}

/* 不可选 */
.option-disabled {
  border-left: 3px solid #858585;
  opacity: 0.6;
}

/* 推荐选项 */
.option-recommended {
  border-left: 3px solid #dcdcaa;
  background-color: rgba(220, 220, 170, 0.05);
}

/* 危险选项 */
.option-dangerous {
  border-left: 3px solid #c72e2e;
  background-color: rgba(199, 46, 46, 0.05);
}
```

### 4.2 选项效果预览
- 显示修为变化
- 显示资源消耗
- 显示寿命影响
- 显示道具需求

### 4.3 选项分组
- 按类型分组（修炼、探索、交易等）
- 使用分隔线区分
- 添加分组标题
