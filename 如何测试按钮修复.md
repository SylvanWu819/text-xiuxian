# 如何测试按钮修复（v2.2.2）

## ⚠️ 重要提示
按钮无响应的问题已修复，但需要**正确安装和测试**才能看到效果。

## 📦 安装步骤

### 方法 1：从 VSIX 文件安装（推荐）

1. **卸载旧版本**
   - 打开 VSCode
   - 按 `Ctrl+Shift+X` 打开扩展面板
   - 找到"修仙模拟器"
   - 右键点击 → 卸载
   - **重启 VSCode**（重要！）

2. **安装新版本**
   - 打开 VSCode
   - 按 `Ctrl+Shift+X` 打开扩展面板
   - 点击右上角的 `...` 菜单
   - 选择"从 VSIX 安装..."
   - 选择 `cultivation-simulator-2.2.2.vsix` 文件
   - 等待安装完成
   - **重启 VSCode**（重要！）

3. **验证安装**
   - 打开扩展面板
   - 找到"修仙模拟器"
   - 确认版本号是 **2.2.2**

### 方法 2：开发模式测试

1. **打开项目**
   ```bash
   cd D:\WordGame
   code .
   ```

2. **编译代码**
   - 按 `Ctrl+Shift+B` 选择"npm: compile"
   - 或在终端运行：`npm run compile`

3. **启动调试**
   - 按 `F5` 启动扩展开发主机
   - 会打开一个新的 VSCode 窗口（标题显示"[扩展开发主机]"）

4. **在新窗口中测试**
   - 在新窗口的侧边栏找到修仙模拟器图标
   - 点击打开

### 3. 测试按钮

#### 测试"开始新游戏"按钮
1. 点击"开始新游戏"按钮
2. 控制台应该显示：`✅ 开始新游戏按钮被点击`
3. 界面应该切换到角色创建界面

#### 测试"继续游戏"按钮
1. 点击"继续游戏"按钮
2. 控制台应该显示：`✅ 继续游戏按钮被点击`
3. 应该显示通知："继续游戏功能开发中"

#### 测试"玩法说明"按钮
1. 点击"玩法说明"按钮
2. 控制台应该显示：`✅ 玩法说明按钮被点击`
3. 应该弹出玩法说明面板

## ❌ 如果按钮仍然无响应

### 检查 1：确认版本
在控制台运行：
```javascript
document.querySelector('.welcome-version').textContent
```
应该显示：`"v2.2.2"`

### 检查 2：确认按钮存在
在控制台运行：
```javascript
console.log('btn-new-game:', document.getElementById('btn-new-game'));
console.log('btn-continue-game:', document.getElementById('btn-continue-game'));
console.log('btn-how-to-play:', document.getElementById('btn-how-to-play'));
```
应该看到三个按钮元素，不是 `null`。

### 检查 3：手动绑定测试
在控制台运行：
```javascript
document.getElementById('btn-new-game').addEventListener('click', () => {
  alert('按钮工作！');
});
```
然后点击按钮，应该弹出提示框。

### 检查 4：查看错误
在控制台的"Console"标签中查看是否有红色的错误信息。

## 🆘 仍然有问题？

### 完全清理重装
1. 卸载扩展
2. 关闭所有 VSCode 窗口
3. 删除 VSCode 扩展缓存：
   - Windows: `%USERPROFILE%\.vscode\extensions\`
   - 找到 `sylvanwu819.cultivation-simulator-*` 文件夹并删除
4. 重启电脑（可选但推荐）
5. 重新安装扩展

### 检查 VSCode 版本
- 打开 VSCode
- 帮助 → 关于
- 确认 VSCode 版本 >= 1.60.0

### 提供调试信息
如果问题仍然存在，请提供：
1. VSCode 版本号
2. 扩展版本号（应该是 2.2.2）
3. 控制台的完整日志（截图或复制文本）
4. 是否有任何错误信息
