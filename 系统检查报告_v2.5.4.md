   - ❌ 没有背包容量限制
   - ❌ 没有道具堆叠上限
   - 影响：可能导致数据膨胀

## 二、发现的Bug

### 🔴 严重Bug

#### Bug #1: 道具需求不消耗道具
**位置**: `src/game/OptionSystem.ts` 和 `src/game/GameEngine.ts`

**问题描述**:
- 选项有 `requirements.requiredItems` 时，只检查是否拥有
- 执行选项后不会消耗这些道具
- 玩家可以用一个道具无限次触发需要该道具的选项

**复现步骤**:
1. 获得一个"灵草"道具
2. 选择需要"灵草"的选项（如炼丹）
3. 选项执行成功，但灵草仍在背包中
4. 可以再次选择同一选项

**影响**: 游戏平衡性严重破坏

**修复方案**:
```typescript
// 在 GameEngine.applyOptionEffects() 中添加
// 消耗选项所需的道具
if (option.requirements?.requiredItems) {
  for (const itemId of option.requirements.requiredItems) {
    // 尝试从不同类型中移除
    if (!this.resourceManager.removeItem(itemId, 1)) {
      if (!this.resourceManager.removePill(itemId, 1)) {
        this.resourceManager.removeArtifact(itemId, 1);
      }
    }
  }
}
```

#### Bug #2: 初始道具可能未正确初始化
**位置**: `src/game/GameEngine.ts` - `createInitialPlayerState()`

**问题描述**:
- 初始化时给了 `healing_pill` 和 `spirit_herb`
- 但这些道具的定义可能不在 `ResourceManager` 中
- 可能导致显示为"未知道具"

**修复方案**:
- 确保所有初始道具都在 `ResourceManager.initializeResourceDefinitions()` 中定义
- 或者在 `UIRenderer.getItemName()` 中添加这些道具的名称映射

### 🟡 中等Bug

#### Bug #3: 背包面板默认隐藏
**位置**: `media/main.js` - `UIRenderer.renderDetailedStats()`

**问题描述**:
- 背包信息在 `detailed-stats` 中，默认 `display: none`
- 需要点击"详细面板"按钮才能看到
- 新玩家可能不知道有背包系统

**修复方案**:
- 将背包信息提升到主面板
- 或者默认展开详细面板
- 或者添加背包快捷按钮

#### Bug #4: 道具名称映射不完整
**位置**: `media/main.js` - `UIRenderer.getItemName()`

**问题描述**:
- 只定义了约20种道具名称
- 但游戏中可能有更多道具ID
- 未映射的道具显示为原始ID

**修复方案**:
- 补全所有可能的道具名称
- 或者从后端传递道具定义到前端

### 🟢 轻微Bug

#### Bug #5: 道具数量显示不直观
**位置**: `media/main.js` - `UIRenderer.renderDetailedStats()`

**问题描述**:
- 道具数量用 `×N` 显示
- 在小屏幕上可能不够清晰

**修复方案**:
- 改为 `数量: N` 或 `(N个)`

   // 新增方法
   private consumeRequiredItem(itemId: string): void {
     if (!this.resourceManager.removeItem(itemId, 1)) {

         this.resourceManager.removeArtifact(itemId, 1);
       }
     }
   }
