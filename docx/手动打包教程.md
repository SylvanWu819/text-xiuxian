# 🎁 手动打包教程

由于 vsce 工具安装失败，这里提供手动打包的方法。

## 方法 1：修复 npm 后使用 vsce（推荐）

### 步骤 1：清理 npm 缓存

```bash
# 清理 npm 缓存
npm cache clean --force

# 删除 node_modules
Remove-Item -Recurse -Force node_modules

# 重新安装依赖
npm install
```

### 步骤 2：全局安装 vsce

```bash
npm install -g @vscode/vsce
```

### 步骤 3：打包

```bash
vsce package --allow-missing-repository --allow-star-activation
```

## 方法 2：使用在线打包服务

访问 VSCode 官方打包服务（如果有的话），或者：

1. 将代码推送到 GitHub
2. 使用 GitHub Actions 自动打包
3. 下载生成的 .vsix 文件

## 方法 3：在另一台电脑上打包

如果你有另一台电脑或虚拟机：

1. 复制整个项目文件夹
2. 在新环境中运行：
   ```bash
   npm install
   npm run compile
   npx @vscode/vsce package
   ```
3. 将生成的 .vsix 文件复制回来

## 方法 4：使用 F5 开发模式（最简单）

**强烈推荐！** 不需要打包，直接运行：

1. 按 F5
2. 新窗口打开
3. 点击左侧太极图标
4. 开始玩游戏！

这样你可以：
- ✅ 立即玩游戏
- ✅ 修改代码后按 Ctrl+Shift+F5 重新加载
- ✅ 不需要每次都打包
- ✅ 适合开发和测试

等游戏完善后，再考虑打包发布。

## 方法 5：请朋友帮忙打包

如果你有朋友的电脑 npm 环境正常：

1. 将项目文件夹压缩发给朋友
2. 朋友运行：
   ```bash
   npm install
   npm run compile
   npx @vscode/vsce package
   ```
3. 朋友将生成的 .vsix 文件发回给你

## 🔧 修复 npm 环境的方法

### 方法 A：重新安装 Node.js

1. 卸载当前的 Node.js
2. 从官网下载最新版：https://nodejs.org/
3. 重新安装
4. 重启电脑
5. 再试一次打包

### 方法 B：使用 nvm 管理 Node.js

```bash
# 安装 nvm-windows
# 下载：https://github.com/coreybutler/nvm-windows/releases

# 安装不同版本的 Node.js
nvm install 18.17.0
nvm use 18.17.0

# 再试一次打包
npx @vscode/vsce package
```

### 方法 C：使用管理员权限

```bash
# 以管理员身份运行 PowerShell
# 然后执行：
npm install -g @vscode/vsce
vsce package
```

## 📝 打包成功后

生成文件：`cultivation-simulator-1.0.0.vsix`

### 安装方法：

```bash
# 命令行安装
code --install-extension cultivation-simulator-1.0.0.vsix
```

或在 VSCode 中：
1. 打开扩展面板（Ctrl+Shift+X）
2. 点击右上角 "..." 菜单
3. 选择"从 VSIX 安装..."
4. 选择 .vsix 文件
5. 重启 VSCode
