# 探索系统优化 v2.3.0

## 问题分析

### 用户反馈
**"事件触发频率太低了。外出探索一定要有反馈才行，可以是事件或者什么都没发现，但得有反馈。"**

### 原有问题
1. **触发率太低**：外出探索只有30%概率触发事件
2. **70%无反馈**：没触发事件时只显示"平静的一天"
3. **体验不佳**：玩家感觉探索没有意义，浪费时间

---

## 解决方案

### 1. **提高探索事件触发率**
- 原触发率：30%
- 新触发率：**80%**
- 提升幅度：**+167%**

### 2. **新增小事件系统**
当没触发大事件时（20%概率），触发小事件：
- 资源发现（30%）
- 修炼感悟（25%）
- 遇到路人（20%）
- 发现线索（15%）
- 什么都没发现（10%）

### 3. **确保每次探索都有反馈**
- 80%：触发大事件（NPC、机缘、危机等）
- 20%：触发小事件（资源、感悟、路人等）
- **100%有反馈！**

```typescript
// 探索时提高触发率
const isExploring = payload.actionId === 'explore';
const eventTriggerRate = isExploring ? 0.8 : 0.3;

const randomEvent = this.gameEngine.tryTriggerRandomEvent(eventTriggerRate);
if (randomEvent) {
  // 触发大事件
} else if (isExploring) {
  // 触发小事件
  const minorEvent = this.generateMinorExplorationEvent();
}
```

## 小事件设计原则

### 1. **快速反馈**
- 事件描述简洁
- 选项清晰明了
- 奖励立即可见

### 2. **合理收益**
- 小事件收益低于大事件
- 但高于"什么都没发生"
- 保持游戏平衡

### 3. **多样性**
- 每类事件有多个变体
- 随机选择，增加新鲜感
- 避免重复感

### 4. **沉浸感**
- 描述贴合修仙世界观
- 观云、听风、观水等自然感悟
- 增强代入感
