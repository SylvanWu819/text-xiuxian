### 1. 选项ID重复问题（非bug）
**发现**: 2个选项ID在不同事件中重复使用
- `retreat_quickly` (出现在 forbidden_zone 和 beast_tide)
- `refine_pill` (出现在 pill_refinement_chance 和 test_item_requirement)

**验证结果**: ✅ **不是问题**
- GameEngine的executeTurn方法使用正确的作用域隔离
- 事件选项优先：先在currentEvent.options中查找
- 日常选项其次：如果不是事件选项，再查找日常选项
- 不同事件的选项不会冲突，因为只有当前事件的选项才会被检查

**代码验证**:
```typescript
// 1. 首先检查是否是当前事件的选项
if (this.currentEvent && this.currentEvent.options) {
  const eventOption = this.currentEvent.options.find((opt: any) => opt.id === optionId);
  if (eventOption) {
    // 处理事件选项
    return { success: true, ... };
  }
}

// 2. 执行玩家选择的日常选项
let option = this.optionSystem.getCachedOption(optionId);
```
