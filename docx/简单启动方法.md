# 🎮 最简单的启动方法

## 你现在看不到图标的原因

你在**开发环境**中，插件还没有被激活。就像你写了一个程序但还没运行一样。

## ✅ 解决方法：按 F5

这是**最简单、最快**的方法！

### 步骤：

1. **按 F5 键**（或点击菜单：运行 → 启动调试）

2. **等待几秒**，会自动打开一个新的 VSCode 窗口
   - 窗口标题会显示"[扩展开发主机]"
   - 这是专门用来测试插件的窗口

3. **在新窗口中**，左侧活动栏会出现"修仙模拟器"图标（太极图案）

4. **点击图标**，游戏界面就会在侧边栏打开！

## 🤔 为什么要开新窗口？

因为你在**开发模式**：
- **当前窗口** = 开发环境（写代码的地方）
- **新窗口** = 测试环境（运行插件的地方）

就像：
- 当前窗口 = 厨房（做菜的地方）
- 新窗口 = 餐厅（吃饭的地方）

## 💡 不想开新窗口？

那需要**安装插件**到当前 VSCode，但这需要：

1. 打包成 .vsix 文件（需要修复 npm 环境）
2. 安装 .vsix 文件
3. 重启 VSCode

**这比按 F5 复杂多了！**

## 🎯 推荐做法

1. **先按 F5 试玩游戏**
   - 看看游戏好不好玩
   - 测试所有功能
   - 修改游戏内容

2. **确定满意后再打包安装**
   - 打包成 .vsix
   - 安装到当前 VSCode
   - 就可以在当前窗口玩了

## 📝 按 F5 后的完整流程

```
1. 按 F5
   ↓
2. VSCode 开始编译和启动
   （左下角会显示"正在启动调试"）
   ↓
3. 新窗口打开
   （标题显示"[扩展开发主机]"）
   ↓
4. 左侧活动栏出现太极图标
   ↓
5. 点击图标
   ↓
6. 游戏界面在侧边栏打开
   ↓
7. 开始玩游戏！🎮
```

## ⚡ 快捷操作

- **F5**：启动插件
- **Ctrl+Shift+F5**：重启插件（修改代码后）
- **Shift+F5**：停止插件

## 🐛 如果按 F5 没反应

1. 检查左下角是否显示错误
2. 查看"终端"面板是否有错误信息
3. 确保代码已编译：`npm run compile`
4. 查看 `.vscode/launch.json` 是否存在

## 🎉 总结

**最简单的方法就是按 F5！**

- ✅ 不需要打包
- ✅ 不需要安装
- ✅ 不需要修复 npm
- ✅ 直接就能玩

**只是会开一个新窗口而已，这很正常！**

---
