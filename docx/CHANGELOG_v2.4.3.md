# 修仙模拟器 v2.4.3 更新日志

## 发布日期
2026-05-01

## 新增功能

### 🔄 事件去重系统
1. **完全阻止事件重复**
   - 已触发过的事件不会再次出现
   - 避免玩家反复遇到相同事件
   - 提升游戏体验和新鲜感

2. **智能重置机制**
   - 当所有事件都触发过时，自动重置记录
   - 允许事件重新触发，避免无事件可触发
   - 确保游戏可以持续进行

3. **事件统计功能**
   - 新增`getEventStats()`方法
   - 可查看总事件数、已触发数、剩余数
   - 便于调试和了解游戏进度

4. **游戏重置支持**
   - 重开游戏时自动重置事件记录
   - 新游戏开始时重置事件记录
   - 确保每次游戏都是全新体验

### 🔧 技术实现
- 修改`EventGenerator.tryTriggerEvent()`方法
- 完全过滤已触发事件，而非降低概率
- 新增`resetTriggeredEvents()`方法
- 新增`getTriggeredEventCount()`方法
- 新增`isEventTriggered()`方法
- 在`GameEngine.initializeGame()`中重置事件
- 在`GameEngine.resetGame()`中重置事件

### 📊 工作原理
```
1. 玩家探索 → 触发事件A
2. 记录事件A已触发
3. 下次探索 → 过滤掉事件A
4. 只从未触发的事件中选择
5. 所有事件都触发后 → 自动重置
6. 重新开始循环
```

### 🎮 游戏体验提升
- 不再重复遇到相同事件
- 每次探索都有新鲜感
- 鼓励玩家多次探索
- 发现所有事件内容

### 💡 设计考虑
- **防止卡死**：所有事件触发后自动重置
- **新游戏体验**：每次重开都是全新开始
- **调试友好**：提供统计信息便于开发

## 日志输出
- 触发事件时输出日志：`[EventGenerator] 触发事件: xxx, 已触发事件数: n`
- 重置记录时输出日志：`[EventGenerator] 所有事件都已触发过，重置事件记录`
- 手动重置时输出日志：`[EventGenerator] 已触发事件记录已重置`
