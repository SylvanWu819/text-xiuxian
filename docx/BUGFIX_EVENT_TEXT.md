# Bug修复：事件结束后文本显示问题

## 版本
v1.0.2

## 问题描述
当事件触发并选择选项后，事件描述仍然保留在界面上，但可选行动已经变成了日常选项，导致文本和选项不匹配。

## 问题原因
事件结束后，代码只清除了事件显示并生成新选项，但没有提供日常状态的描述文本，导致界面显示空白或保留旧事件文本。

## 修复方案
在 `src/extension.ts` 中添加 `generateDailyEvent()` 方法，当没有随机事件触发时，生成日常状态描述：

- 根据当前季节和修为境界生成不同的日常描述
- 炼气期、筑基期、金丹期、元婴期等不同境界有不同的描述文本
- 每个境界有多条随机描述，增加游戏趣味性

## 修改文件
- `src/extension.ts` - 添加 `generateDailyEvent()` 方法
- `package.json` - 版本号更新为 1.0.2

## 测试方法
1. 安装新版本扩展
2. 开始新游戏
3. 触发一个事件（如"上古洞府"）
4. 选择事件选项
5. 确认事件结束后显示日常状态描述，而不是保留旧事件文本

## 安装
```bash
code --install-extension cultivation-simulator-1.0.2.vsix
```
